Client Side

< grid-directive 
	class="myExampleGrid"
	grid="gridData"
	rowselect="onClientRowSelect(row)"
	pagesize="100"
	pagepos="both"
	view="grid"
	columncount = 2
	switchview=false>

Attribute usage

class : The css class to control the styling

grid : The data which grid will take to construct the grid. Following is the sample format for the same

this.$scope.gridData = {

rowselect : Here you can provide a callback method name on which you will get the selected row object

pagesize : The total number of records you want to show per page

pagepos : This is used to declare the position of pagination. Acceptable values are:

'top' : to show on top 'bottom' : to show on bottom 'both' : to show on top and bottom

view : This tells how you want to do the initial display of a grid. Acceptable values are:

"grid" : To show in tabular view "list" : To show in list view

columncount : This is used for list view. You can declare how many items you want to display in each row

switchview: To enable or disable the switcher to toggle the view of the data in list or grid view. Acceptable values are :

false: if you want to hide it true: if you want to display it

Server Side

< grid-directive 
	class = "myExampleGrid"
	grid = "serverGridData"
	rowselect = "onServerRowSelect(row)"
	callserver = "callServer(params)"
	pagesize = "1000"
	pagepos = "both"
	view = "grid"
	server = "true"
	params = "params"
	columncount = 3
	switchview = true>

Attribute usage

class : The css class to control the styling

grid : The data which grid will take to construct the grid. Following is the sample format for the same

Following will be the format from REST call

{
  "gridData" : [],   // the rows object
  "currentPage" : 1, // which is the current page
  "records" : 200, // Total number of records
  "pageSize": 100 // records showing per page
}

this.$scope.gridData = {

		},

			width:'25%'
		},
		birthday: {

			id: 'birthday',

			width:'25%'
		},
		salary: {
			label: 'Salary',
			id: 'salary',
			sort: true,
			width:'25%'
		}
	},
	grid : [],  // should come from REST response and populate here
	totalRecords : 200 // should come from REST response and populate here
	pageSize : 100 // should come from REST response and populate here
	currentPage : 1 // should come from REST response and populate here
};

Major requirement

You need to broadcast the server response in above object using $broadcast like below:

context.$scope.$broadcast('griddata', {grid: context.$scope.serverGridData});

rowselect : Here you can provide a callback method name on which you will get the selected row object

callserver : Here you can provide a callback method to interact with server for various client interaction like, search, sorting, pagination etc. The updated params will be provided as an argument to this method

pagesize : The total number of records you want to show per page

pagepos : This is used to declare the position of pagination. Acceptable values are:

'top' : to show on top 'bottom' : to show on bottom 'both' : to show on top and bottom

view : This tells how you want to do the initial display of a grid. Acceptable values are:

"grid" : To show in tabular view "list" : To show in list view

server: To let grid know that user interaction will be server side

"false": if the grid is not server "true": if the grid is server

params : This will be updated each time when user interact with features like pagination, sorting, search etc. Acceptable format is:

this.$scope.params = {
	page : 0,  // page number to send to server
	sortKey : 'age', // Key for sorting
	sortDirection: 'asc', // direction of sorting, Acceptable values are 'asc' or 'desc'
	search : null // search string
};

columncount : This is used for list view. You can declare how many items you want to display in each row

switchview: To enable or disable the switcher to toggle the view of the data in list or grid view. Acceptable values are :

false: if you want to hide it true: if you want to display it
